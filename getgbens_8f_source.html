6 C> ABSTRACT: FIND AND UNPACK A GRIB MESSAGE.
7 C> READ A GRIB INDEX FILE (OR OPTIONALLY THE GRIB FILE ITSELF)
8 C> TO GET THE INDEX BUFFER (I.E. TABLE OF CONTENTS) FOR THE GRIB FILE.
9 C> (THE INDEX BUFFER IS SAVED FOR USE BY FUTURE PROSPECTIVE CALLS.)
10 C> FIND IN THE INDEX BUFFER A REFERENCE TO THE GRIB MESSAGE REQUESTED.
11 C> THE GRIB MESSAGE REQUEST SPECIFIES THE NUMBER OF MESSAGES TO SKIP
12 C> AND THE UNPACKED PDS AND GDS PARAMETERS. (A REQUESTED PARAMETER
13 C> OF -1 MEANS TO ALLOW ANY VALUE OF THIS PARAMETER TO BE FOUND.)
14 C> IF THE REQUESTED GRIB MESSAGE IS FOUND, THEN IT IS READ FROM THE
15 C> GRIB FILE AND UNPACKED. ITS MESSAGE NUMBER IS RETURNED ALONG WITH
16 C> THE UNPACKED PDS AND GDS PARAMETERS, THE UNPACKED BITMAP (IF ANY),
17 C> AND THE UNPACKED DATA. IF THE GRIB MESSAGE IS NOT FOUND, THEN THE
18 C> RETURN CODE WILL BE NONZERO.
19 C> THIS OBSOLESCENT VERSION HAS BEEN REPLACED BY GETGBE.
