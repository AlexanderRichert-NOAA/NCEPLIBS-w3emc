SUBPROGRAM: W3AI41 CONSTANT SIZE BINARY STRING PACKER PRGMMR: ALLARD, R.

ORG: W342 DATE: 80-04-01

ABSTRACT: PACKS CONSTANT SIZE BINARY STRINGS INTO AN ARRAY. THIS PACKING REPLACES BITS IN THE PART OF THE OUTPUT ARRAY INDICATED BY THE OFFSET VALUE. W3AI40 IS THE REVERSE OF W3AI41. (SEE W3AI32 TO PACK VARIABLE SIZE BINARY STRINGS.)

PROGRAM HISTORY LOG: 80-04-01 R.ALLARD (ORIGINAL AUTHOR) ASMEMBLER LANGUAGE VERSION. 84-07-05 R.E.JONES RECOMPILED FOR NAS-9050 89-11-04 R.E.JONES WROTE FORTRAN VERSION OF W3AI40 TO PACK CONSTANT SIZE BINARY STRINGS 89-11-05 R.E.JONES CONVERT TO CRAY CFT77 FORTRAN 98-03-10 B. VUONG REMOVE THE CDIR$ INTEGER=64 DIRECTIVE

USAGE: CALL W3AI40 (KFLD,KOUT,KLEN,KNUM,KOFF)

INPUT: KFLD - INTEGER INPUT ARRAY OF RIGHT ADJUSTED STRINGS KLEN - INTEGER NUMBER OF BITS PER STRING (0 < KLEN < 33) KNUM - INTEGER NUMBER OF STRINGS IN 'KFLD' TO PACK KOFF - INTEGER NUMBER SPECIFYING THE BIT OFFSET OF THE FIRST OUTPUT STRING. THE OFFSET VALUE IS RESET TO INCLUDE THE LOW ORDER BIT OF THE LAST PACKED STRING OUTPUT: KOUT - INTEGER OUTPUT ARRAY TO HOLD PACKED STRING(S)

EXIT STATES: ERROR - KOFF < 0 IF KLEN HAS AN ILLEGAL VALUE OR KNUM < 1 THEN KOUT HAS NO STRINGS STORED.

EXTERNAL REFERENCES: NONE

REMARKS: THIS SUBROUTINE SHOULD BE WRITTEN IN ASSEMBLER LANGUAGE. THE FORTRAN VERSION RUNS TWO OR THREE TIMES SLOWER THAN THE ASEMBLER VERSION. THE FORTRAN VERSION CAN BE CONVERTED TO RUN ON OTHER COMPUTERS WITH A FEW CHANGES. THE BIT MANIPULATION FUNCTIONS ARE THE SAME IN IBM370 VS FORTRAN 4.1, MICROSOFT FORTRAN 4.10, VAX FORTRAN. MOST MODERN FORTRAN COMPILER HAVE AND, OR, SHIFT FUNCTIONS. IF YOU ARE RUNNING ON A PC, VAX AND YOUR INPUT WAS MADE ON A IBM370, APOLLO SUN, H.P.. ETC. YOU MAY HAVE TO ADD MORE CODE TO REVERSE THE ORDER O BYTES IN AN INTEGER WORD. NCAR SBYTES CAN BE USED INSTEAD OF THIS SUBROUTINE. PLEASE USE NCAR SBYTES SUBROUTINE INSTEAD OF THIS SUBROUTINE.

ATTRIBUTES: LANGUAGE: CRAY CFT77 FORTRAN MACHINE: CRAY Y-MP8/832
